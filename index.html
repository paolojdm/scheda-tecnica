Introduction

JavaScript è un linguaggio di scripting multipiattaforma orientato agli oggetti. È un linguaggio piccolo e leggero. All'interno di un ambiente host (ad esempio, un browser web), JavaScript può essere collegato a oggetti del suo ambiente su cui fornire il controllo programmatico loro.

JavaScript contiene una libreria standard di oggetti, come Array, Data e matematica e un insieme fondamentale di elementi linguistici come operatori, strutture di controllo e dichiarazioni. JavaScript di base può essere esteso per una varietà di scopi integrandolo con oggetti aggiuntivi; Per esempio:

  • JavaScript lato client estende il linguaggio principale fornendo oggetti per controllare un browser e il suo Document Object Model (DOM). Ad esempio, le estensioni lato client consentono l'inserimento di un'applicazione elementi su un modulo HTML e rispondere agli eventi dell'utente come il mouse clic, input del modulo e navigazione della pagina.
  • JavaScript lato server estende il linguaggio principale fornendo oggetti rilevanti per l'esecuzione di JavaScript su un server. Per esempio, le estensioni lato server consentono a un'applicazione di comunicare con un file database, fornire continuità di informazioni da un'invocazione a un'altra dell'applicazione o eseguire manipolazioni di file su a server.

JavaScript and Java

JavaScript e Java sono simili in qualche modo ma fondamentalmente diverso in alcuni altri. Il linguaggio JavaScript assomiglia a Java ma non dispone della tipizzazione statica di Java e del controllo avanzato del tipo. JavaScript segue la maggior parte della sintassi delle espressioni Java e delle convenzioni di denominazione e costrutti di base del flusso di controllo che era il motivo per cui lo era rinominato da LiveScript a JavaScript.

In contrasto con il sistema di classi in fase di compilazione di Java creato da dichiarazioni, JavaScript supporta un sistema runtime basato su un piccolo numero di tipi di dati che rappresentano numerici, booleani e stringhe valori. JavaScript ha un modello a oggetti basato su prototipi invece di modello a oggetti basato su classi più comune. Il modello basato sul prototipo fornisce l'ereditarietà dinamica; cioè, ciò che viene ereditato può variare per i singoli oggetti. JavaScript supporta anche funzioni senza eventuali requisiti dichiarativi particolari. Le funzioni possono essere proprietà di oggetti, eseguiti come metodi liberamente tipizzati.

JavaScript è un linguaggio in formato molto libero rispetto a Java. Non devono dichiarare tutte le variabili, classi e metodi. Tu non hai preoccuparsi se i metodi sono pubblici, privati o protetto e non è necessario implementare interfacce. variabili, parametri e i tipi restituiti dalla funzione non sono tipizzati in modo esplicito.

Variables

Le variabili vengono utilizzate come nomi simbolici per i valori nell'applicazione. I nomi delle variabili, chiamati identificatori, sono conformi a determinate regole.

Un identificatore JavaScript deve iniziare con una lettera, un carattere di sottolineatura (_) o simbolo del dollaro ($); i caratteri successivi possono anche essere cifre (0-9). Poiché JavaScript distingue tra maiuscole e minuscole, le lettere includono i caratteri da "A" a "Z" (maiuscole) e i caratteri da "a" a "z" (minuscolo).

È possibile utilizzare lettere ISO 8859-1 o Unicode come å e ü identificatori. Puoi anche utilizzare le sequenze di escape Unicode come caratteri negli identificatori. Alcuni esempi di nomi legali sono Number_hits, temp99 e _name.

Declaring variables
Puoi dichiarare una variabile in tre modi:

Con la parola chiave var. Per esempio, var x = 42. Questa sintassi può essere utilizzata per dichiarare sia variabili locali che globali.

Assegnandogli semplicemente un valore. Per esempio, x = 42. Questo dichiara sempre una variabile globale. Genera un severo avviso JavaScript. Non dovresti usare questa variante.

Con la parola chiave let. Per esempio, let y = 13. Questa sintassi può essere utilizzata per dichiarare una variabile locale con ambito blocco. Vedere l'ambito variabile di seguito.

Variable scope

Quando dichiari una variabile al di fuori di qualsiasi funzione, viene chiamata variabile globale, perché è disponibile per qualsiasi altro codice nel documento corrente. Quando dichiari una variabile all'interno di una funzione, viene chiamata variabile locale, perché è disponibile solo all'interno di quella funzione.

JavaScript prima di ECMAScript 2015 non ha un ambito per le istruzioni di blocco; piuttosto, una variabile dichiarata all'interno di un blocco è locale alla funzione (o all'ambito globale) in cui risiede il blocco. Ad esempio, il codice seguente registrerà 5, perché l'ambito di x è la funzione (o il contesto globale) all'interno del quale viene dichiarato x, non il blocco, che in questo caso è un'istruzione if.

if (true) { var x = 5; } console.log(x); // 5

Questo comportamento cambia quando si utilizza la dichiarazione let introdotta in ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Global variables

Le variabili globali sono infatti proprietà dell'oggetto globale. Nelle pagine web l'oggetto globale è window, quindi puoi impostare e accedere alle variabili globali utilizzando la sintassi window.variable.

Di conseguenza, potete accedere alle variabili globali dichiarate in una finestra o frame da un'altra finestra o frame specificando il nome del frame della finestra. Ad esempio, se in un documento viene dichiarata una variabil denominata phoneNumber, puoi fare riferimento a questa variabile da un iframe come parent.phoneNumber.

Constants

È possibile creare una costante denominata di sola lettura con la parola chiave const. La sintassi di un identificatore costante è la stessa di un identificatore variabile: deve iniziare con una lettera, un carattere di sottolineatura o un segno di dollaro e può contenere caratteri alfabetici, numerici o di sottolineatura.

const PI = 3.14;

Una costante non può modificare il valore tramite l'assegnazione o essere dichiarata nuovamente mentre lo script è in esecuzione. Deve essere inizializzato su un valore.

Le regole di ambito per le costanti sono le stesse di quelle per le variabili di ambito let block. Se la parola chiave const viene omessa, si presuppone che l'identificatore rappresenti una variabile.

Non è possibile dichiarare una costante con lo stesso nome di una funzione o di una variabile nello stesso ambito. Per esempio:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements } However, object attributes are not protected, so the following statement is executed without problems. const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
Data types

L'ultimo standard ECMAScript definisce sette tipi di dati:

  • Sei tipi di dati che sono primitivi:

    • Boolean. true and false.
    • nullo. Una parola chiave speciale che denota un valore nullo. Poiché JavaScript fa distinzione tra maiuscole e minuscole, null non è la stessa cosa di Null, NULL o qualsiasi altra variante.
    • undefined. Una proprietà di primo livello il cui valore è undefined.
    • Number. 42 o 3.14159.
    • String. "Howdy"
    • Symbol (new in ECMAScript 2015).Un tipo di dati le cui istanze sono univoche e immutabili.
  • e Object
Sebbene questi tipi di dati siano una quantità relativamente piccola, consentono di eseguire funzioni utili con le proprie applicazioni. Oggetti e funzioni sono gli altri elementi fondamentali del linguaggio. Puoi pensare agli oggetti come contenitori con nome di valori e alle funzioni come procedure che la tua applicazione può eseguire.
if...else statement
Utilizzare l'istruzione if per eseguire un'istruzione se una condizione logica è vera. Utilizzare la clausola opzionale else per eseguire un'istruzione se la condizione è falsa. Un'istruzione if appare come segue: if (condition) { statement_1; } else { statement_2; } condition can be any expression that evaluates to true or false. See Boolean for an explanation of what evaluates to true and false. If condition evaluates to true, statement_1 is executed; otherwise, statement_2 is executed. statement_1 and statement_2 can be any statement, including further nested if statements.

Puoi anche comporre le istruzioni usando else if per testare più condizioni in sequenza, come segue:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } In the case of multiple conditions only the first logical condition which evaluates to true will be executed. To execute multiple statements, group them within a block statement ({ ... }) . In general, it's good practice to always use block statements, especially when nesting if statements: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } It is advisable to not use simple assignments in a conditional expression, because the assignment can be confused with equality when glancing over the code. For example, do not use the following code: if (x = y) { /* statements here */ } If you need to use an assignment in a conditional expression, a common practice is to put additional parentheses around the assignment. For example: if ((x = y)) { /* statements here */ }
while statement
Un'istruzione while esegue le sue istruzioni finché una condizione specificata risulta vera. Un'istruzione while appare come segue: while (condition) statement If the condition becomes false, statement within the loop stops executing and control passes to the statement following the loop.

Il test della condizione si verifica prima dell'esecuzione dell'istruzione nel ciclo. Se la condizione restituisce true, l'istruzione viene eseguita e la condizione viene nuovamente testata. Se la condizione restituisce false, l'esecuzione si interrompe e il controllo viene passato all'istruzione while successiva.

Per eseguire più istruzioni, utilizzare un'istruzione di blocco ({ ... }) per raggruppare tali istruzioni.

Example:

Il seguente ciclo while esegue l'iterazione finché n è inferiore a tre:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

With each iteration, the loop increments n and adds that value to x. Therefore, x and n take on the following values:

  • After the first pass: n = 1 and x = 1
  • After the second pass: n = 2 and x = 3
  • After the third pass: n = 3 and x = 6

After completing the third pass, the condition n < 3 is no longer true, so the loop terminates.

Function declarations
A function definition (also called a function declaration, or function statement) consists of the function keyword, followed by:
  • The name of the function.
  • A list of arguments to the function, enclosed in parentheses and separated by commas.
  • The JavaScript statements that define the function, enclosed in curly brackets, { }.

For example, the following code defines a simple function named square:

function square(number) { return number * number; }

La funzione quadrato accetta un argomento, chiamato numero. La funzione consiste in un'istruzione che dice di restituire l'argomento della funzione (ovvero il numero) moltiplicato per se stesso. L'istruzione return specifica il valore restituito dalla funzione.

return number * number;

I parametri primitivi (come un numero) vengono passati alle funzioni in base al valore; il valore viene passato alla funzione, ma se la funzione modifica il valore del parametro, questa modifica non si riflette a livello globale o nella funzione chiamante.
